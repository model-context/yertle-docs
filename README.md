# Yertle Docs

The source for the Yertle developer documentation site at [docs.yertle.com](https://docs.yertle.com).

This repo contains the OpenAPI spec and the hand-written guides that make up the public docs. It's rendered by [Mintlify](https://mintlify.com).

## Repo structure

```
yertle-docs/
├── mint.json              # Mintlify config (theme, navigation, OpenAPI link)
├── openapi.json           # Auto-generated OpenAPI spec from the Yertle backend
├── favicon.png            # Site favicon
├── logo/turtle.png        # Site logo
├── introduction.mdx       # Landing page
├── quickstart.mdx         # Quickstart guide
├── authentication.mdx     # Authentication guide
├── concepts.mdx           # Core concepts (Git-like model)
├── tools/
│   ├── cli.mdx            # Yertle CLI overview
│   └── mcp.mdx            # Yertle MCP server overview
├── api-reference/
│   └── overview.mdx       # Reference landing (rest auto-generated by Mintlify)
└── LICENSE
```

## Local preview

```bash
npm i -g mintlify
mintlify dev
```

Opens the docs at `http://localhost:3000` with hot reload.

## Updating the OpenAPI spec

The `openapi.json` file is generated from the Yertle backend. To regenerate it:

```bash
cd ../yertle/backend
pipenv run python scripts/export_openapi.py --output ../../yertle-docs/openapi.json
```

Then commit and push the updated spec. Mintlify will rebuild the API reference automatically.

## Deploy

Pushes to `main` are deployed automatically by the Mintlify GitHub app. There is no manual deploy step.
